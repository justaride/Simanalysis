Rapport og presentasjon / 15. juni 2026

Simanalysis

Et lokalt Sims 4 Doctor-verktøy som analyserer Mods, forklarer risiko, hjelper brukeren å teste trygt, og gjør filendringer bare når de er eksplisitte, journalførte og reversible.

Produktet har beveget seg fra konflikt-scanner til en bredere sikkerhetsflate for mod-samlinger: inventory, Doctor, Treatment, Patch Day, Save/Tray/Cache-beskyttelse og Update Desk.

Status
v3
Ikke “ferdig produkt” ennå. Men fundamentet er nå sterkt: lokal sannhet, eksplisitte planer, manifest-baserte handlinger, og rollback som produktregel.

Produktfortelling

Fra mod-kaos til kontrollerte valg.

Simanalysis er et desktop- og CLI-verktøy for spillere og skapere som bruker Sims 4 mods og custom content. Det leser lokale filer før spillet starter, bygger evidens, og foreslår neste trygge steg.

Les Scanner Mods, Tray, saves, cache og staging uten å endre filer.
Forstå Parser DBPF, ressursnøkler, STBL, SimData, logs og historikk.
Forklar Viser konflikter, krasjbevis, patch-risiko og filsignaler.
Stage Lager planer og manifester før noe flyttes eller kopieres.
Rollback Handlinger skal kunne reverseres fra journalen, ikke gjettes.
Analyze first, stage second, mutate third, rollback always.

Nåværende kapabiliteter

Et sikkerhetslag rundt hele Sims 4-mappen.

Mod-analyse

Scanner `.package` og `.ts4script`, hasher filer, leser DBPF-ressurser, finner tuning/resource-overlapp og eksporterer strukturert analyse.

Doctor

Kombinerer script- og UI-exception autopsy, timeline, ledger-kontekst, verdicts og playbooks uten å mutere Sims-filer.

Treatment

Auto-Bisect med manifest-first flyttinger, restore, verdict logging, handoff og game-process/path/symlink guards.

Inventory Ledger

SQLite-basert lokal sannhet: scans, snapshots, ressurser, historikk, endringer siden sist og per-fil events.

Operating Table

Cleanup-planer kan stages, applyes og restores via manifest. Ingen sletting, ingen overskriving, ingen handling mens spillet kjører.

Trust Protectors

Patch Day, Profile Manager Light, Cache Doctor, Save Protector og Tray Protector er read-only eller app-state-only sikkerhetsflater.

Flater brukeren møter

CLI for presisjon. Desktop for trygg gjennomgang.

Simanalysis har samme sikkerhetskontrakter gjennom Python-core, bridge, Tauri og React. Det gjør at CLI, tester og desktop ikke trenger å være tre forskjellige sannheter.

Den viktigste produktbeslutningen er at “vis meg status” og “gjør en filhandling” er separate nivåer. Read-only kommer først. Mutasjon krever eksplisitt plan, manifest og rollback.

Desktop

Dashboard, Mods, Inventory, Cleanup, Doctor, Treatment, Live Monitor, Patch Day, Cache, Save, Tray, Update Desk og Settings.

CLI

`doctor`, `bisect`, `ledger`, `ops`, `patch-day`, `cache`, `save-protector`, `tray`, og `updates` for scriptbar drift.

Bridge

NDJSON sidecar-kontrakter binder desktop til samme backend-logikk som testene og CLI-en bruker.

Hva har utviklet seg de siste ukene

Fra scanner til operasjonssikkert lokalt system.

Uke 1
True Engine og real corpus

Falske ressurskonstanter ble erstattet med S4TK-verifisert registry, real-fixture kontrakt kom inn, og parser-output ble mer ærlig.

Uke 2
Ledger og Cleanup foundation

SQLite inventory, snapshots, event-log og read-only Cleanup Planner ga prosjektet lokal sannhet og trygge planflater.

Uke 3
Operating Table og Doctor gates

Manifest-first cleanup apply/restore, Doctor CLI, bisection handoff, replay gates, timeline og ledger-kontekst ble samlet.

Uke 4
Trust polish

Patch Day Shield, Profile Manager Light, Cache Doctor, Save Protector og Tray Protector flyttet produktet nærmere en trygg hverdagsapp.

Nå
Update Desk

Staged downloads kan klassifiseres, planlegges og nå også commit/undo på løse `.package` og `.ts4script` med hash, symlink, path, collision, game-process og crash-recovery guards. Desktop er fortsatt preview-only.

Produktprinsipper

Trygghet er ikke pynt. Det er arkitekturen.

Hard boundaries

  • Ingen automatisk permanent sletting av brukerfiler.
  • Ingen filmutasjon mens The Sims 4 kjører.
  • Ingen path escapes, symlinkede operasjonsmål eller stille overskriving.
  • Ingen “unknown mod is safe”-påstander etter patch.

Trust mechanics

  • Read-only status først.
  • Plan og snapshot før handling.
  • Manifest/journal før copy eller move.
  • Undo/restore basert på journalert evidens.

Dette gjør Simanalysis mer konservativ enn mange “rydd opp for meg”-verktøy. Den konservatismen er et produktfortrinn: Sims-brukere skal føle at appen hjelper dem å forstå, ikke at den tar eierskap over samlingen deres.

Roadmap til ferdig produkt

Fire etapper fra dagens v3 til public-ready.

1. Close v3 Safety

  • Desktop Update Desk install/apply-kontroller.
  • Trygg arkivinstallering med traversal-, symlink- og corrupt-archive gates.
  • Flere crash-recovery og property-test scenarier.
  • Browser-QA for alle nye desktop-operasjoner.

2. Trust Polish

  • Cache-clearing som manifestert quarantine, ikke delete.
  • Patch Day re-enable workflow bundet til profiler.
  • Save Protector v1 med profile/save-sammenheng.
  • Tray dependency signals med tydelig usikkerhet.

3. Intelligence

  • Classification v1: CAS, Build/Buy, UI, script, gameplay, default replacement.
  • Conflict Engine v2 med override-intensjon og profile awareness.
  • Script Security Analyzer som statisk risikosignal, aldri runtime execution.
  • Exception Analyzer v2 for flere loggformater.

4. Release Package

  • SBOM, signering/notarisering og sikkerhetsgjennomgang.
  • Ærlig README, changelog og “not EA/Maxis official” branding.
  • Installer/distribusjon for desktop.
  • Public v3 smoke test på frisk maskin og real-world corpus.

Definition of done

Når er Simanalysis “ferdig nok” for Public v3?

Produktkrav

  • Brukeren kan forstå Mods-helse uten terminal.
  • Alle muterende workflows har preview, explicit approval og undo.
  • Patch/update/cache/save/tray-flater har samme trust language.
  • Appen er ærlig om begrensninger og ukjent kompatibilitet.

Releasekrav

  • Python, Rust, web og real-corpus gates grønt.
  • Desktop QA uten horisontal overflow på desktop og mobil-bredder.
  • Signert/notarisert desktop build der plattformen krever det.
  • README og changelog matcher faktisk shipped surface.

Kort sagt: Public v3 bør lanseres når Simanalysis trygt kan si “jeg kan hjelpe deg å rydde, teste og oppdatere” uten å skjule at brukeren fortsatt er beslutningstakeren.

Oppsummering

Simanalysis er blitt et lokalt kontrollrom for Sims 4 mods.

De siste ukene har prosjektet fått den typen fundament som skiller et nyttig hobbyverktøy fra et produkt brukere kan stole på: verifisert parser-sannhet, lokal ledger, read-only reviewflater, manifest-first handlinger og en konsekvent filosofi om at brukerfiler ikke er noe man “fikser” uten sporbarhet.

Neste fase handler ikke om å legge til mest mulig. Den handler om å lukke Update Desk trygt, polere trust-flater, bevise releasepakken, og bare deretter åpne for mer intelligente integrationslag.
